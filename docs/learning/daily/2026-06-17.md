### タスク 1-2: ボタンコンポーネントのスタイル共通化（途中）

#### 共通化の設計検討

- 各ボタン（Edit / Save / Done / Delete / Cancel）の共通点・差異を自分で分析できた
  - 共通: `<button class="icon-btn">` の HTML 構造、クリックイベント、同じクラス名
  - 差異: アイコン・アイコンサイズ・細かい CSS・クリック時の処理

#### slot vs props の使い分け

- アイコンは `<slot>` で渡す判断をした → 型処理の煩雑さを避けられる・親でアイコンを直接書いた方が可読性が高い
- `@click` の処理は最初「子で持つ」と考えたが、`Base` コンポーネントに全ボタンの処理を持たせるのは責務が広すぎると気づいた → 親で `@click` を渡す（Vue のフォールスルー属性）

#### scoped CSS の詳細度問題にぶつかった

- `ButtonIcon.vue` の `<style>`（scoped なし）と各ボタンの `<style scoped>` で詳細度が異なることを実装して発見した
- scoped CSS は内部的に `[data-v-xxx]` が付くため詳細度が上がり、グローバル CSS を常に上書きする
- 「設計が間違っていたら作り直す」という判断を自分でできた

#### CSS 変数を props 経由で渡すアプローチに切り替え

- `:style` バインディングで CSS カスタムプロパティを渡し、スタイルの差異を表現する方法を採用
- 既存の `var(--color-success)` などの CSS 変数は文字列としてそのまま渡せることを確認
- `transition` を props で渡す際、オブジェクトにしようとしたが「CSS は文字列で渡す方がシンプル」という判断に至った
- デフォルト値は `??` 演算子で設定し、props をオプショナルにした

#### ここまでの実装状況

- `ButtonIcon.vue` を `components/layouts/` に作成（CSS 変数 + slot ベース）
- `DoneButton.vue`・`EditButton.vue` を `ButtonIcon` に移行済み
- 残り: `SaveButton` / `DeleteButton` / `CancelButton`
